## Internal deep review

### Syscall migration — `src/alsa.cyr`, `src/error.cyr`, `src/mixer.cyr`, programs, tests

73 raw `syscall(N, ...)` sites migrated:

| From | To | Sites | Wrapper provenance |
|------|----|-------|--------------------|
| `syscall(1, ...)` | `sys_write(...)` | 35 | stdlib `lib/syscalls_*_linux.cyr` |
| `syscall(2, ...)` | `sys_open(...)` | 3 | stdlib (aarch64 routes to `openat(AT_FDCWD, ...)`) |
| `syscall(3, ...)` | `sys_close(...)` | 2 | stdlib |
| `syscall(16, ...)` | `syscall(SYS_IOCTL, ...)` | 20 | constant per-arch (29 on aarch64, 16 on x86_64) — no wrapper exists in stdlib |
| `syscall(60, ...)` | `sys_exit(...)` | 11 | stdlib |
| `syscall(228, ...)` | (left raw) | 2 | x86_64-only programs (`throughput.cyr`, `latency_test.cyr`); no aarch64 cross-build target |

Verification:
- ✓ x86_64 build clean (no behavior change — wrappers compile to identical syscalls)
- ✓ aarch64 cross-build clean (was the goal — produces valid ARM ELF)
- ✓ 258/258 tests pass (no regression from the migration)
- ✓ All 6 silent real-HW programs PASS — `audio_open_playback` chain through `sys_open` lands at `/dev/snd/pcmC{N}D{M}p` exactly as before
- ✓ Distlib diff-clean against the v0.9.0 header (the migration touched dist content only via the source bundle concatenation)

The two `syscall(228, ...)` (`clock_gettime`) sites in
`programs/throughput.cyr` / `programs/latency_test.cyr` are
documented as x86_64-only inline. CI's `Cross-build aarch64`
step only targets `programs/smoke.cyr`, so these don't gate
the aarch64 release artifact. If a future cut needs aarch64
real-HW timing, add a small portable `_clock_monotonic()`
helper to the program (or upstream `sys_clock_gettime` to
stdlib).

### Supply chain — `[deps.patra]` git override

Same shape as the existing `[deps.yukti]` override:
- `cyrius.lock` SHA256-pins each resolved file (now includes
  patra at 1.9.2's hash).
- `cyrius deps --verify` step in CI rejects any drift.
- "Lock file present" CI step asserts the lock exists before
  resolving (the v0.3.0 LOW-1 fix).
- `cyrius.cyml` comment block documents the removal trigger.

One operational note caught during this sweep: `~/.cyrius/deps/`
is a persistent local cache. A stale `~/.cyrius/deps/patra/1.9.2/`
directory (populated with 1.9.0 content from a prior aborted
resolve) silently misled the build until `rm -rf` of the cache
and a fresh resolve. The lockfile guard catches drift between
lock and lib; it does NOT catch a cache that's labeled correctly
but populated incorrectly. Reported as an upstream
`cyrius deps` hardening candidate (not a vani-side fix).

## Regression assertions

The migration is its own regression: the CI aarch64 cross-build
step now FAILS the run if any future raw `syscall(2, ...)` /
`syscall(3, ...)` etc. lands in vani code. CPU tests don't need
new assertions — the wrapper functions are stdlib-vetted and
the existing 258 assertions cover the call paths.

The pre-existing `test_ioctl_size_encoding_matches_struct_size`
and `test_ctl_ioctl_size_encoding` tests now also pin `SYS_IOCTL`
indirectly through the migrated ioctl call paths — if a future
toolchain releases break the `SYS_IOCTL` constant on either arch,
the suite trips.
